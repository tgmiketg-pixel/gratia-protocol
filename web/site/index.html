How Gratia Works

Three steps. No special hardware. No upfront cost. Just your phone.

01

Live Your Life

Proof of Life runs silently in the background. Unlock your phone, carry it, connect to Wi-Fi, charge it at night — normal daily use generates the cryptographic attestation that proves you're a real person with a real phone. Zero battery impact. Zero user action required.

02

Plug In to Mine

Proof of Presence activates when your phone is plugged in and above 80% battery. Mining earns GRAT at a flat rate — every minute of mining pays the same. No diminishing returns. No time-of-day restrictions. Your phone charges first, then mines.

03

Spend & Govern

Send GRAT to anyone — tap phones with NFC, scan a QR code, or paste an address. Vote on protocol changes with one phone, one vote governance. No token-weighted voting. Your voice is equal regardless of your balance.

Phase 3 Complete

The latest milestone brings cross-platform support, scalability, advanced privacy, and offline resilience.

📱

iOS Support

Full iOS app built with SwiftUI, all sensor managers (Core Motion, Core Location, Core Bluetooth, Core NFC), and the same shared Rust core via UniFFI Swift bindings. One codebase, both platforms.

🛰

Bluetooth Mesh

Offline transactions via Bluetooth LE and Wi-Fi Direct. Bridge peers relay mesh transactions to the consensus layer. Works without any internet connection — true peer-to-peer.

🌐

Geographic Sharding

10 geographic shards scale throughput to ~2,000 TPS while keeping each phone's storage requirements low. Cross-shard transactions handled transparently with finality proofs.

🔒

Groth16 ZK Proofs

Complex privacy-preserving smart contract interactions via Groth16 zero-knowledge proofs. Proof generation runs during Mining Mode when the phone has power to spare.

Roadmap

✓

Phase 1 — Proof of Concept

Rust core (14 crates, 725 tests), Android app with wallet/mining/staking/governance, phone-to-phone consensus over internet, NFC tap-to-transact, Bulletproofs ZK, block explorer. Two phones mining real blocks across the internet.

✓

Phase 2 — Smart Contracts & Sync

GratiaScript compiler + WASM execution (interpreter + Wasmer), linear memory, contract storage, event emission, state sync protocol, block sync between peers, debug bypass hardened for release builds.

✓

Phase 3 — iOS, Sharding & Mesh

Full iOS app (SwiftUI + same Rust core), geographic sharding (10 shards, ~2,000 TPS), Bluetooth mesh + Wi-Fi Direct offline transactions, Groth16 ZK proofs for complex smart contracts, environmental oracle layer, 15 Rust crates, 866 tests.

Phase 4 — Public Mainnet

Security audit by external firm, App Store submission (Google Play + Apple App Store), geographic sharding activation (10 shards live), multi-party ceremony for Groth16 trusted setup, community governance launch.
